Value Mapping

Each raster Symbol has a property value_selector. This will be used to retrieve a value from the rwo record belonging to the geometry that is drawn. At the moment this is restricted to numerical values only.

There is only one implementation and no subclasses. The Smart raster Symbol uses the pixel space of the raster_image to draw a vertical bar at a size relative to a maximum scale. This scale is calculated over all the unique rwo_records that are in the geometry set passed to the plugin. This can be set manually, in which case no calculations are done.

The module raster_symbols contains a raster image with the name "bar2.png". Its a half transparent 20x100 pixels. The raster definition DIAGRAM-BAR can be used as an example.

 

•

The style for the vertical bar representing is the fill style of the goto_area_style property.

•

The value of an instance (rwo_record) is drawn at the top right of the vertical bar. The text style used for this text is the highlight_text_style.

•

If the value of the property draw_text_string? is True, it will draw the external name and the name of the value_selector at the position as specified in the symbol. In case this is a field with a defined unit, the unit string is drawn between brackets.

•

If the value scale is set, either in the symbol, or at the statistical engine, this value is shown at the top of the raster Symbol.

 

 
In the screen recording below, you can see how it works. With the Smallworld Explorer all Leaks are retrieved from the database. They all are selected and the button  Goto is pressed. It shows the default Goto mapping for a Water Office object (with the value for :rs_string).

The location of the leak is added to the Specific Mappings, mapped to the symbol DIAGRAM-BAR. On redraw, it appears that the default symbol is still drawn. This is because the browser sends a goto request with all geometries. In The RWO Hierarchy list, all other geometries than Location are selected and mapped to the NOTHING symbol. A refresh shows the bar2.png.

Then the value_selector volume is added and the value of draw_text_string? set to Yes. The next refresh of the screen shows the final result.
