The modules

The Raster Symbols source files are organized as bitmap resources of modules. On retrieval of resources, customisation modules are included.
This allows filtering, authorisation, and proper caching. It also allows the creation of Definitions of symbols when the module is loaded.
Raster Symbols come with a separate product for testing: raster_symbols_test. See Unit tests for more details.
 

Raster symbols do belong to a sw_module

The Raster Symbols have the property filename and module_name to (re)create them. Both configuration and the source bitmap (png) files can be organised to be loaded on demand.
The product contains several of those specialisations, e.g. the module rs_cam_db_raster_symbols and wo_raster_symbols.

The list of modules

raster_symbols_core

Core classes raster_symbol_handler, raster_symbol_plugin, core resources, help etc. This module contains everything to make it work.

raster_symbols_gui

Contains the Configuration Interface

raster_symbols

The module intended as main resource for 'bitmaps' and configurations. Contains a lot of examples.

raster_symbols_custom

Customisation product that contains a custom module for raster_symbols
