Design considerations

There are many icon packages on the internet, but you might want to develop your own.

Purpose

It is a good idea to envisage the purpose of icons. Are they drawn always, or only in specific situations? Text drawing is important, because it tells the user the what is where.

Pixel size

If you use a raster image with unequal width and height, the thumbnails in the Configuration GUI will be distorted, because it forces the image in a size of 16x16 pixel. The gain in performance when drawing a lesser amount of pixels is (in the case of a raster type symbol) not noticeable.

Style (family)

The colour and style of Raster Symbols should have a similar look and feel and be evident in itself.

Transparency

If possible, use alpha channel transparency.
