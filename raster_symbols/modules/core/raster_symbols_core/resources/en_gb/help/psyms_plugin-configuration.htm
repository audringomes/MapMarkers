Plugin Configuration

To embed the MapMarkers in an application, the following xml configuration is used:
The folder ../modules/raster_symbols_core/resources/base/data/application_integration contains two template files:

- application_config.xml for integration with the application config;
- embedded_map_gui.xml for adding the toolbar to the Map Gui.

Add plugin to application

The file application_config.xml contains the code that add MapMarkers as plugin to the application:

<plugins>
<!--  START RASTER SYMBOLS -->

 
<plugin name="raster_symbols" class_name="raster_symbol_plugin">
    <property name="config_definition_file_name" value="config.xml"/>
    <property name="config_definition_module_name" value="raster_symbols_core"/>
    <property name="sticky_goto?" value="true"/>
    <property name="goto_skip_invisble_geoms?" value="false"/>
    <property name="use_default_for_goto?" value="true"/>
    <property name="no_draw_labels?" value="false"/>
</plugin>

 

<!-- Addition of Options Panel to the Options plugin -->
  <plugin name="options" class_name="options_plugin">
    <panels>
        <!-- -->
        <panel name="raster_symbols.user_options" />
    </panels>
  </plugin>
<!--  END RASTER SYMBOLS -->

 
</plugins>

Add toolbar to GUI

The file embedded_map_gui.xml contains an example of how to add the toolbar to the Map Gui:

<?xml version="1.0" standalone="yes"?>
<gui>
  <!-- TOOLBARS -->
  <!-- START RASTER SYMBOLS -->
  <toolbar name="rsy_full_toolbar">
    <action source="application" name="raster_symbols.use_goto_raster_symbols"/>
    <action source="application" name="raster_symbols.sticky_goto"/>
    <action source="application" name="raster_symbols.goto_skip_invisible"/>
    <action source="application" name="raster_symbols.goto_use_default"/>
    <action source="application" name="raster_symbols.no_draw_labels?"/>
    <action source="application" name="raster_symbols.no_draw_geoms?"/>

     <action source="application" name="raster_symbols.tick"/>
    <action source="application" name="raster_symbols.activate_management_interface"/>
    <action source="application" name="raster_symbols.help"/>
  </toolbar>
  <!-- END RASTER SYMBOLS -->
 
  <dock name="top">
    <toolbar name="navigation"/>
    <toolbar name="objects"/>
    <!-- START RASTER SYMBOLS -->
    <toolbar name="rsy_full_toolbar"/>
    <!-- END RASTER SYMBOLS -->
    <section/>
    <toolbar name="interaction"/>
    <toolbar name="map_modes"/>
    <toolbar name="refresh"/>
    <toolbar name="view"/>
  </dock>
 
  <dock name="bottom">
    <toolbar name="trail"/>
    <toolbar name="probes"/>
    <toolbar name="trail_measurer"/>
  </dock>
 
  <work_area>
    <paned_window dynamic="no" border="yes">
        <self width="600" height="600" />
    </paned_window>
  </work_area>
 
</gui>

 

 

Auto load config

To load a configuration on the start of an application, a special xml file can be provided. The name and location of this file is configurable in the plugin by the properties config_definition_file_name and config_definition_module_name. The default is config.xml in the raster_symbols_core module.

<?xml version="1.0" encoding="ISO-10646-UCS-2" standalone="yes"?>
<config>
<configuration name="raster_symbols" autoload="true" overwrite="true">
  <module name="raster_symbols"/>
  <file name="raster_symbols_configuration.xml"/>
</configuration>
</config>

This file is read when the plugin is initiated. One or more <configuration> tags can be defined. It reads the configuration and will load the defined Raster Symbol configuration files with symbols, mappings and properties if the value for the attribute autoload is true. If the attribute overwrite is true, it will replace existing symbols, mapping and options.
